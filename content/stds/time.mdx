# time

## Overview
The `time` standard library provides basic functionalities for handling time-related operations, including getting the current time, timestamp conversion, and time formatting.

## Types

### time_t
Time struct, used to represent a point in time and related information.

```nature
type time_t = struct {
    i64 sec     // Seconds-level timestamp
    i64 nsec    // Nanoseconds part
    rawptr<tm_t> tm // Pointer to local time struct
}
```

## Functions

### now
Creates a `time_t` instance based on the current time.

```nature
fn now():time_t
```

Example:
```nature
var t = time.now()
println(t.datetime())  // Output: 2024-01-01 12:00:00
```

### unix
Gets the current Unix timestamp (seconds-level).

```nature
fn unix():i64
```

Example:
```nature
var timestamp = time.unix
println(timestamp)  // Output: 1704067200
```

## time_t Methods

### timestamp
Gets the seconds-level timestamp.

```nature
fn time_t.timestamp():i64
```

### ms_timestamp
Gets the milliseconds-level timestamp.

```nature
fn time_t.ms_timestamp():i64
```

### ns_timestamp
Gets the nanoseconds-level timestamp.

```nature
fn time_t.ns_timestamp():i64
```

### datetime
Formats the time as a readable date-time string (format: YYYY-MM-DD HH:mm:ss).

```nature
fn time_t.datetime():string
```

## Usage Example

```nature
import time

fn main() {
    // Get current time
    var t = time.now()

    // Get timestamps of different precisions
    println(t.timestamp())     // Seconds-level timestamp
    println(t.ms_timestamp())  // Milliseconds-level timestamp
    println(t.ns_timestamp())  // Nanoseconds-level timestamp

    // Format time
    println(t.datetime())      // E.g.: 2024-01-01 12:00:00

    // Get current timestamp directly
    println(time.unix())
}
```

## Notes
1. The time string returned by the `datetime()` method uses the local timezone.
2. Timestamp-related methods all return UTC time.
3. This standard library relies on system calls and the C standard library to implement underlying functionalities.
