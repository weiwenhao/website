# fmt

## Overview
The `fmt` standard library provides string formatting functionality, supporting formatted output for various data types.

## Functions

### sprintf

Generates a formatted string based on the format string and arguments.

```nature
fn sprintf(string format, ...[any] args):string
```

### printf

Formats and prints a string to standard output.

```nature
fn printf(string format, ...[any] args)
```

### scannf

Reads a formatted string from standard input.

```nature
fn scannf(string format,...[any] args)
```

## Format Specifiers
Directives in the format string begin with `%`, supporting the following formats:

### Basic Formats
- `%v` - General format, automatically selects the appropriate format based on the value's type.
- `%d` - Integer format.
- `%s` - String format.
- `%c` - Character format.
- `%%` - Outputs a percent sign %.

### Width and Padding
Supports specifying the minimum output width and padding character:
- `%Nd` - Specifies minimum width N (padded with spaces).
- `%0Nd` - Specifies minimum width N (padded with 0s).

## Supported Types
- Integer types: `int`, `i8`, `i16`, `i32`, `i64`, `uint`, `u8`, `u16`, `u32`, `u64`
- Floating-point types: `float`, `f32`, `f64`
- Boolean type: `bool`
- String: `string`
- Pointers: `ptr`, `void_ptr`, `raw_ptr`, `vec`, `map`, `set`, `tup`

## Usage Example

```nature
import fmt

fn main() {
    // Basic formatting
    fmt.printf("Hello %s\n", "World")     // Output: Hello World
    fmt.printf("Number: %d\n", 42)         // Output: Number: 42

    // Width and padding
    fmt.printf("%05d\n", 123)              // Output: 00123
    fmt.printf("%5d\n", 123)               // Output:   123

    // General formatting
    fmt.printf("Bool: %v\n", true)         // Output: Bool: true
    fmt.printf("Float: %v\n", 3.14)        // Output: Float: 3.140000

    // Character
    fmt.printf("Char: %c\n", 65 as u8)     // Output: Char: A

    // Format as string
    var s = fmt.sprintf("Value: %v", 42)
    fmt.printf("%s\n", s)                  // Output: Value: 42
}
```

## Notes
1. When format specifier and argument types do not match, an error marker will be output (e.g., `%!d`, `%!s`, etc.).
2. When the number of arguments is insufficient, it will output `%!` followed by the corresponding format character.
3. Unsupported format specifiers will output `%?` followed by the corresponding character.
4. Floating-point formatting defaults to retaining 6 decimal places.
